# 测试用例生成器

系统化地发现测试盲点，不遗漏边界、异常、状态组合。

## 核心能力

1. **盲点扫描**：基于代码/需求，找出容易被忽略的测试场景
2. **多维度覆盖**：功能路径、边界值、异常、性能、安全
3. **可执行输出**：直接生成测试代码框架
4. **覆盖率检查**：提供覆盖率验证清单

## 快速生成流程

### Step 1: 理解被测对象

用户提供以下任一：
- 功能需求描述
- 接口定义（API 文档）
- 代码片段
- 已有测试（找缺口）

**快速提问**（如果需要）：
1. 输入数据的范围/类型是什么？
2. 有哪些外部依赖（数据库/API/文件）？
3. 成功和失败的状态分别是什么？

### Step 2: 系统化生成测试用例

按照 **FBEPS-I 模型** 生成：

| 维度 | 说明 | 占比 |
|-----|------|------|
| **F**unctional | 功能路径测试 | 30% |
| **B**oundary | 边界值测试 | 20% |
| **E**xception | 异常/错误测试 | 20% |
| **P**erformance | 性能/压力测试 | 10% |
| **S**ecurity | 安全测试 | 5% |
| **I**ntegration | 功能交互/关联测试 | 15% |

> **⚠️ 重要：Integration 维度**
> 
> 很多 Bug 发生在**功能与功能的交互处**，而非单个功能内部。务必检查：
> - 功能 A 的操作是否影响功能 B 的数据/状态
> - 跨功能的业务流程是否完整
> - 共享资源的竞争和冲突
> - 功能调用链的异常传递

## 测试盲点检查清单

生成用例后，用此清单检查是否有遗漏：

### 输入维度
- [ ] Null / undefined / nil
- [ ] 空字符串 / 空数组 / 空对象
- [ ] 空白字符（空格、tab、换行）
- [ ] 极值（0, -1, 最大整数）
- [ ] 特殊字符（emoji、中文、阿拉伯文）
- [ ] 超长长度的输入
- [ ] 错误的数据类型

### 状态维度
- [ ] 初始状态
- [ ] 中间状态
- [ ] 终态
- [ ] 无效状态转换
- [ ] 状态回滚

### 时间维度
- [ ] 时区处理
- [ ] 闰年/闰秒
- [ ] 超时场景
- [ ] 定时任务边界（23:59:59）

### 并发维度
- [ ] 重复提交
- [ ] 并发修改同一资源
- [ ] 读写并发
- [ ] 分布式锁

### 依赖维度
- [ ] 依赖服务超时
- [ ] 依赖服务返回错误
- [ ] 依赖服务返回异常数据格式
- [ ] 网络抖动/中断

### 功能交互维度（⚠️ 重点检查）
- [ ] **功能依赖**：功能B依赖功能A的数据/状态，测试A未完成/异常时B的行为
- [ ] **数据流向**：功能A修改的数据是否正确反映到功能B的读取结果中
- [ ] **状态联动**：功能A的状态变更是否触发功能B的相应变化
- [ ] **并发冲突**：两个功能同时操作同一资源时的竞争条件
- [ ] **顺序敏感**：功能A→B执行 vs B→A执行的结果是否一致
- [ ] **级联影响**：功能A失败对依赖它的功能B、C、D的影响
- [ ] **回滚一致性**：跨多个功能的操作回滚后，各功能状态是否一致
- [ ] **共享资源**：配置、缓存、文件等共享资源的读写一致性
- [ ] **跨功能事务**：涉及多个功能的数据操作是否满足ACID
- [ ] **流程完整性**：完整业务流程中各功能节点的衔接和数据传递
- [ ] **边界功能**：系统边界处的功能交互（如导入→校验→存储）
- [ ] **权限联动**：功能A的权限变更是否影响功能B的访问

| 场景 | 命名示例 | 说明 |
|-----|---------|------|

| 大版本更新 | `CASE-v2.0.md`、`CASE-v3.0.md` | 功能重构/测试策略变更 |

### 文件写入要求

1. **写入位置**：当前工作目录或用户指定的目录
2. **操作步骤**：
   - 使用 WriteFile 工具将完整报告写入文件
   - 文件写入成功后，向用户展示文件路径和简要概览
   - 告知用户可以在文件中查看完整的测试用例
